Identity
CLI Tool Name
vb (verbose)
What It Is
Multi-Lens Terminal Knowledge Base — a vault-based personal reference system queryable entirely from the terminal.
What It Does
vb is a terminal-native knowledge base where you store and query technical information through topic + lens combinations. You give it a noun like disk or ssh and a flag like --why, --arch, --cli-tools, or --importance and it returns exactly that dimension of knowledge — nothing more, nothing tangential.

Knowledge lives as structured markdown files inside a vault — a folder you initialize with vb init, organized by category and topic. Every topic has its own folder containing flag-mapped files like WHY.md, ARCH.md, CLI_TOOLS.md. You author directly in your editor of choice, no GUI, no special interface.

The more you feed it — especially with --used, your personal command log per topic — the more it becomes your single terminal command for answering "what is this, why does it exist, how does it work, what do I use for it, and what exact command did I use last time." No browser, no navigation, no context switch. Just ask and it answers.
Vault Structure

Flag System — Lenses
--why
→ WHY.md · WHY_GUI.md
Why this topic exists as a concept. Prose format. Answers the foundational "why does this thing exist at all" before you touch it.
--importance
→ IMPORTANCE.md
Why it matters, what breaks without it, what depends on it. Different from --why — this is the consequence lens, not the origin lens.
--cli-tools
→ CLI_TOOLS.md
Scoped command reference. Only tools directly for this topic — no adjacent concepts. disk means disk tools only, not partition or filesystem.
--arch
→ ARCH.md · ARCH.mmd
Architecture in terminal tree format. Add --gui to render as a Mermaid diagram in a lightweight webview. The only flag with a visual render target.
--used
→ USED.md
Your personal command log for this topic. The most used flag day-to-day. Solves the "wait what was that command again" problem permanently. Tagged, timestamped, scoped.
--gotchas
→ GOTCHAS.md
Footguns, common mistakes, things that burned you. The flag you'd wish existed every time something breaks unexpectedly mid-workflow.
--refs
→ REFS.md
Source docs, man pages, articles that informed your entries. Always know where the knowledge came from if you need to go deeper.
--gui
→ universal modifier
Composable with any flag. --why --gui renders markdown. --arch --gui renders Mermaid diagram. GUI is always optional — terminal is the default, GUI is earned.
Tech Stack
Language · Foundation

Go

Compiles to a single static binary with zero runtime dependencies. No Node.js, no Bun, no Python — just ship vb, vb.exe, vb-macos. Fast startup, low memory, and the strongest ecosystem for CLI tooling that exists today. Cross-platform by default.

Core CLI Structure
Cobra
core · cli
Industry standard for Go CLIs. Handles the vb --arch disk command/flag pattern cleanly — subcommands, flags, aliases, auto-generated help. Used by kubectl, Hugo, and GitHub CLI.
Viper
core · config
Pairs with Cobra for config management. Reads .vb/config.toml so vb remembers your vault path, preferred editor, theme, and lint preferences without re-typing them every session.
Terminal Output & Styling
Glamour
format · render
Renders markdown as styled terminal output — headers, code blocks, lists, tables. Makes vb --why disk output look polished instead of raw text. Each flag type gets its own render style.
Lip Gloss
format · styling
Terminal styling library. Controls colors, borders, padding per flag output. --arch tree looks different from --why prose — every lens has its own visual identity in terminal.
Goldmark
format · parser
Markdown parser for Go. Parses your .md vault files and feeds them into Glamour for rendering. Extensible AST — used for the custom schema linter per flag type.
Linting & Validation
Custom Schema Linter
format · linter
Built by you — a rule engine per flag file type. CLI_TOOLS.md expects command blocks. WHY.md expects prose. ARCH.md expects tree nodes. USED.md expects command + description pairs. Runs on vb lint <topic> or automatically on save if lint_on_save = true in config.
GUI Render (--gui modifier)
Wails or lorca
gui · webview
Lightweight Go webview — not Electron, no bloat. Spawned only when --gui is passed. Renders Mermaid diagrams for --arch --gui and clean markdown for --why --gui. Opens, shows, closes. That's it.
Editor Integration & Distribution
os/exec (stdlib)
tooling · editor
Go's built-in process spawner. Launches micro, nano, vim, or $EDITOR when you open a vault file for editing. Zero dependency — pure stdlib.
goreleaser
tooling · distribution
Automates cross-platform binary builds. One command produces vb-linux-amd64, vb-darwin-arm64, vb-windows-amd64.exe — ready to distribute or install via package managers.
cobra-completion
tooling · dx
Built into Cobra. Generates shell completions for bash, zsh, fish, PowerShell. vb --<TAB> autocompletes flags. vb disk<TAB> autocompletes topic names from the vault index.
Project Insights & Specifications
Vault Initialization
vb init creates the .vb/ folder in the current directory — identical behavior to git init. Without it, vb has nothing to query. The vault is the source of truth. Multiple vaults can exist on one machine for different contexts (work, personal, homelab).
File-to-Flag Mapping is Exact
Flag names map directly to filenames. --why → WHY.md, --arch → ARCH.md, --cli-tools → CLI_TOOLS.md, --used → USED.md. No ambiguity, no magic. If the file doesn't exist, vb tells you it hasn't been authored yet. Authoring is always done directly in your editor — no special interface.
--used is the Daily Driver Flag
vb disk --used 'lsblk -f' -d "show all filesystems on disk" appends to USED.md with the command, description, and timestamp. Query with vb disk --used to list all saved commands for that topic. Add --tag to categorize. Use vb --used --tag mount to search across all topics by tag. Solves the permanent "wait what was that command again" problem.
--gui is Always Optional
The terminal is the default and primary interface. --gui is a composable modifier — never required, always available. --arch --gui renders a Mermaid diagram. --why --gui renders clean markdown in a webview. Power users who never leave the terminal ignore it entirely. The tool never forces a mode.
Dot Config — .vb/config.toml
User-configurable persistent settings. Exists because some settings shouldn't be typed as flags every time. Includes: editor (micro, nano, vim, $EDITOR), knowledge_path, theme (dark/light), tree_style (unicode/ascii for --arch), lint_on_save (true/false). The config is the boundary between what the tool assumes and what the user owns.
Scoped Querying — No Tangents
vb disk --cli-tools returns only disk tools. Not partition tools, not filesystem tools — disk only. Scope is enforced by the author at write time through the linter. This is the core philosophy: the flag is a precise lens, not a broad search. You get exactly what you asked for.
Cross-Platform by Design
Single Go binary. No runtime. Works identically on Linux, macOS, Windows — and in SSH sessions, VMs, and containers where GUI tools don't exist. The --gui modifier gracefully degrades if no display is available and falls back to terminal output.
Value Compounds Over Time
Day one: sparse. Month six: irreplaceable. Every time you go down a rabbit hole solving something, you put it in vb. Every --used entry you save is a command you'll never have to re-find. The knowledge base is entirely built from your own experience — nobody else has it, nobody can replicate it. That's the real long-term value.
