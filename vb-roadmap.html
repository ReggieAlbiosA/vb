Development Strategy

vb — Roadmap

Chronological Execution Plan for Agentic AI Build

Phase 01: Scaffolding & Configuration
01
Foundations
Skeleton & Vault Discovery
  • 01
    Project Init: Initialize Go module and setup Cobra/Viper structure. Create the root main.go and basic cmd/ hierarchy.
  • 02
    The init Command: Implement vb init. It must create the .vb/ directory and a default config.toml.
  • 03
    Config Resolver: Ensure Viper correctly finds the .vb/config.toml by searching upwards from the current directory (like Git).
✔ Validation
Run vb init in a temp folder. Verify .vb/config.toml exists. Run vb help to see generated documentation.
Phase 02: Core Querying (The Lenses)
02
Knowledge Engine
File-to-Flag Mapping & Rendering
  • 01
    Topic Resolver: Logic to map vb <topic> to a folder path (e.g., hardware/disk/) using a recursive search or index.json.
  • 02
    Lens Mapping: Implement the flag logic. --why looks for WHY.md, --cli-tools for CLI_TOOLS.md.
  • 03
    Glamour Integration: Pipe the file content through Glamour for terminal markdown rendering.
  • 04
    Lip Gloss Styling: Apply unique borders/colors per lens type to differentiate --arch from --why visually.
✔ Validation
Create hardware/disk/WHY.md manually. Run vb disk --why. Verify output is styled and readable.
Phase 03: The Write Loop
03
Authoring & History
Editor Integration & Command Logging
  • 01
    The Edit Command: Implement vb edit <topic> --flag. Use os/exec to launch the user's $EDITOR.
  • 02
    --used Logic: Implementation of the command logger. Must support appending to USED.md with [TIMESTAMP] Command - Description.
  • 03
    Tagging System: Allow --used --tag <name> to categorize logged commands for future cross-topic search.
✔ Validation
Run vb disk --used 'lsblk' -d 'list blocks'. Verify USED.md is updated correctly. Run vb edit disk --why to ensure editor opens.
Phase 04: Validation & Quality
04
Linter
Markdown Schema Linter
  • 01
    Linter Core: Build the vb lint command using Goldmark AST to inspect Markdown structure.
  • 02
    Lens Rules: Define rules per lens (e.g., CLI_TOOLS.md must contain code blocks, WHY.md must have a header).
  • 03
    Auto-Linter Hook: (Optional) Check config for lint_on_save to run validation after vb edit.
✔ Validation
Purposely break a file format. Run vb lint disk. Verify errors are reported clearly.
Phase 05: The Visual Modifier
05
GUI Target
Mermaid & Webview Rendering
  • 01
    Wails/Lorca Setup: Integrate a lightweight webview bridge.
  • 02
    The --gui Modifier: Detect flag and divert output to the webview instead of terminal stdout.
  • 03
    Mermaid Bridge: For --arch --gui, load ARCH.mmd into an HTML template with Mermaid.js.
✔ Validation
Run vb disk --arch --gui. A window should appear with the rendered diagram. Ensure --gui falls back to terminal if no display is found.
Phase 06: Polish & Distribution
06
Ship It
Completions & Binaries
  • 01
    Shell Completion: Use Cobra to generate .bash, .zsh, and .fish completion scripts.
  • 02
    GoReleaser Config: Setup .goreleaser.yaml for cross-platform binary builds.
  • 03
    Documentation: Generate USAGE.md automatically from Cobra help commands.
✔ Validation
Run goreleaser --snapshot. Verify binaries for Linux, Mac, and Windows are generated in dist/.
